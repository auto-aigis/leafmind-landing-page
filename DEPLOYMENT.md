# LeafMind Waitlist + Auth System - Deployment Guide\n\n## Overview\nThis document provides step-by-step instructions to deploy the LeafMind landing page with waitlist capture, authentication, and analytics integration.\n\n## Prerequisites\n- Supabase project (free tier acceptable)\n- Render.com account\n- Vercel account (for Analytics dashboard)\n- Node.js 18+\n\n## Step 1: Set Up Supabase Database\n\n### 1.1 Create Tables\nIn your Supabase project:\n1. Open the SQL Editor\n2. Copy the entire contents of `frontend/supabase/migrations/init.sql`\n3. Run the SQL to create:\n   - `waitlist` table (email, submitted_at, source)\n   - `users` table (for authentication)\n   - `sessions` table (for session management)\n   - `subscriptions` table (for future payment integration)\n4. Verify Row-Level Security (RLS) is enabled on all tables (should be set by the SQL)\n\n### 1.2 Get Supabase Credentials\nFrom your Supabase dashboard:\n1. Go to **Settings** → **API**\n2. Copy:\n   - `NEXT_PUBLIC_SUPABASE_URL` (public URL)\n   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` (anon key)\n   - `SUPABASE_SERVICE_ROLE_KEY` (service role key - keep secret)\n\n## Step 2: Environment Variables\n\nCreate a `.env.local` file in the project root with:\n\n```\nNEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co\nNEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key\nSUPABASE_SERVICE_ROLE_KEY=your-service-role-key\nNEXT_PUBLIC_API_URL=http://localhost:8000\nSESSION_SECRET=your-random-secret-key\nPLATFORM_WEBHOOK_URL=https://your-webhook-url.com\nPROJECT_ID=your-project-id\nWEBHOOK_SECRET=your-webhook-secret\nNEXT_PUBLIC_VERCEL_ANALYTICS_ID=your-vercel-analytics-id\n```\n\n## Step 3: Local Development\n\n```bash\ncd frontend\nnpm install\nnpm run dev\n```\n\nVisit `http://localhost:3000`\n\n### Test Waitlist Form\n1. Open `http://localhost:3000?source=twitter`\n2. Enter a test email and submit\n3. Verify the record appears in Supabase `waitlist` table with `source='twitter'`\n\n## Step 4: Deploy to Render\n\n### 4.1 Push to GitHub\n```bash\ngit add .\ngit commit -m \"Initial LeafMind setup\"\ngit push origin main\n```\n\n### 4.2 Create Render Deployment\n1. Go to [render.com](https://render.com)\n2. Click **New +** → **Web Service**\n3. Connect your GitHub repository\n4. Configure:\n   - **Name**: leafmind-landing-page\n   - **Environment**: Node\n   - **Build Command**: `npm install && npm run build`\n   - **Start Command**: `npm start`\n   - **Root Directory**: frontend\n\n### 4.3 Add Environment Variables\nIn Render deployment settings, add:\n- `NEXT_PUBLIC_SUPABASE_URL`\n- `NEXT_PUBLIC_SUPABASE_ANON_KEY`\n- `SUPABASE_SERVICE_ROLE_KEY`\n- `SESSION_SECRET`\n- `PLATFORM_WEBHOOK_URL`\n- `PROJECT_ID`\n- `WEBHOOK_SECRET`\n- `NEXT_PUBLIC_VERCEL_ANALYTICS_ID`\n\n### 4.4 Deploy\nClick **Create Web Service** and wait for deployment to complete.\n\n## Step 5: Smoke Test (Critical Gate)\n\n**This is the hard go/no-go for Twitter campaign launch.**\n\n### 5.1 Test Waitlist Submission\n1. Open production URL: `https://leafmind-landing-page.onrender.com?source=twitter`\n2. Submit test email (e.g., `test-twitter-2024@example.com`)\n3. Verify:\n   - ✅ Success message appears: \"You're on the list! We'll be in touch soon 🌿\"\n   - ✅ Browser DevTools Network tab shows `POST /api/waitlist` with 200 status\n\n### 5.2 Verify Supabase Record\n1. In Supabase dashboard, navigate to `waitlist` table\n2. Verify the row exists with:\n   - `email`: test-twitter-2024@example.com\n   - `source`: 'twitter'\n   - `submitted_at`: recent timestamp (UTC)\n3. Duplicate submission shows: \"You're already on the list!\"\n\n### 5.3 Verify Analytics\n1. Go to [Vercel Analytics Dashboard](https://vercel.com/analytics)\n2. Select the LeafMind project\n3. Verify:\n   - Page views are recorded\n   - Custom event `waitlist_signup` is tracked\n\n**✅ If all three checks pass, the Twitter campaign is GO.**\n\n## Step 6: Configure DNS (Optional)\n\nTo use a custom domain:\n1. In Render, go to your Web Service → **Settings**\n2. Add custom domain (e.g., `leafmind.com`)\n3. Update DNS records as instructed by Render\n\n## Troubleshooting\n\n### Waitlist Form Shows Error\n- Check browser console for network errors\n- Verify `SUPABASE_SERVICE_ROLE_KEY` is set in Render env vars\n- Ensure `waitlist` table exists in Supabase\n- Check RLS policy allows service role inserts\n\n### Email Validation Fails\n- Verify `users` table exists\n- Check email is unique (not already in database)\n- Ensure `users` table RLS allows service role access\n\n### Analytics Not Showing\n- Verify `NEXT_PUBLIC_VERCEL_ANALYTICS_ID` is set\n- Check Vercel project is linked to GitHub repository\n- Allow 5 minutes for initial data ingestion\n\n### Session Cookie Not Setting\n- Ensure `SESSION_SECRET` is set in production\n- Verify cookies are enabled in browser\n- Check domain is not on cookie blocklist\n\n## API Endpoints\n\n### Waitlist\n- `POST /api/waitlist` - Submit email + source\n  - Body: `{ email: string, source: string }`\n  - Response: `{ status: \"success\" | \"duplicate\" }`\n\n### Auth\n- `POST /api/auth/register` - Create account\n- `POST /api/auth/login` - Authenticate\n- `POST /api/auth/logout` - Clear session\n- `GET /api/auth/me` - Get current user\n- `POST /api/auth/verify-email` - Verify email token\n- `POST /api/auth/resend-verification` - Resend verification email\n\n## Next Steps\n\n1. **Twitter Campaign**: Once smoke test passes, launch Twitter campaign with `?source=twitter` utm param\n2. **Payment Integration**: Wire Paddle checkout on `/pricing` page\n3. **Email Notifications**: Set up transactional emails for verification and waitlist\n4. **User Dashboard**: Add post-signup features on `/dashboard`\n\n## Support\n\nFor deployment issues:\n1. Check Render logs: `https://dashboard.render.com`\n2. Check Supabase logs: `https://supabase.com/dashboard/projects`\n3. Check Vercel Analytics: `https://vercel.com/analytics`\n"
